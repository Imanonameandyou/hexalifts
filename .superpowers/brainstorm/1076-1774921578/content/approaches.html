Architecture Approaches

How should the Next.js quiz funnel be structured?

A

Single-Page App (SPA) Feel

One page, all client-side. The entire quiz lives in a single Next.js page (/). A React state machine manages the flow: landing → questions → loading → teaser → email gate → results. Smooth transitions between steps, no page reloads. The only server interaction is the /api/capture-email route that hits Klaviyo.

Pros

  • Fastest perceived performance — no navigation between steps
  • Smooth CSS transitions between quiz states
  • Simplest routing — one page component
  • Easy to add progress animations
  • All scoring logic stays in one place

Cons

  • Larger initial JS bundle (all questions loaded upfront)
  • No shareable URLs for individual steps
  • Browser back button needs manual handling
  • SEO only applies to landing state
B

Multi-Page with Dynamic Routes

Separate pages per step. Each quiz step is its own Next.js route: / (landing), /quiz/1 through /quiz/6, /loading, /result/[type]. Scores passed via URL params or cookies. Each page is server-rendered.

Pros

  • Each step has a shareable URL
  • Browser back/forward works natively
  • Smaller per-page bundles
  • Can track analytics per page

Cons

  • Page transitions feel heavier — flash of loading
  • Score state management across pages is awkward
  • More files to manage (10+ page components)
  • Harder to animate between questions smoothly
  • Over-engineered for a linear funnel
C

Hybrid: SPA Quiz + Server Result Page

Best of both. The landing page and quiz (questions 1–6 + loading + teaser + email gate) are a single client-side SPA flow on /. After email capture, redirect to /result/[type] — a server-rendered page with full SEO, shareable URL, and the complete breakdown. This is the only page that needs its own route.

Pros

  • Smooth quiz experience (no page loads during questions)
  • Result page is shareable and SEO-friendly
  • Clean separation: quiz flow vs. result content
  • Result page can be revisited without retaking quiz
  • Natural architecture boundary

Cons

  • Slightly more complex than pure SPA
  • Need to pass result type between pages (URL param)

⬆ Recommended. The quiz is a linear funnel — it should feel like one continuous flow. But the result page has real content worth indexing and sharing.
